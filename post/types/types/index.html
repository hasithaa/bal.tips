Ballerina has a variety of type descriptors that are used to describe types. In this post, I am summarizing those.

Simple Data Values

Nil: ()
boolean
int
float
decimal
💡  Nil Data Type 

The Nil data type represents a single value called nil, which represents the absence of any other value.
  • ()
  • Nil Literal : () or null (for JSON support).
  • Optional Type 𝓣? ⇔ 𝓣|().
  • returns () means Absence of return value in a function.
  • Multiple toString representations.

💡  Boolean Data Type 

The Boolean data type represents boolean truth values: true and false.
  • boolean
  • Boolean Literal, e.g. true & false
  • Lang Library : ballerina/lang.boolean

💡  Integer Data Type 

The Integer data type represents 64-bit signed integer values.
  • int
  • Integer Literal e.g. 10, 12345, 0xAD
  • Lang Library : ballerina/lang.int
  • byte and more subtypes.
  • No implicit conversion between numbers.

🛠  Float Data Type 

The Float data type represents 64-bit IEEE 754-2008 binary floating point numbers.
  • float
  • Floating Point Literal, e.g. 12.34, 0xA1.2B
  • Lang Library : ballerina/lang.float
  • No implicit conversion between numbers.

🛠  Decimal Data Type 

The Decimal basic data type represents 128-bit IEEE 754-2008 decimal floating point values.
  • decimal
  • Floating Point Literal, e.g. 12.34,
  • Lang Library : ballerina/lang.decimal
  • No implicit conversion between numbers.

Sequence Data Values

string
xml
🛠  String Data Type 

The string data type is a representation of a sequence of zero or more Unicode characters.
  • string
  • String Literal
    e.g. "Hello, World...!"
  • String Template
    e.g. string `Hello, ${name}...!`
  • Lang Library : ballerina/lang.string
  • Iterable 🔁

🛠  XML Data Type 

The xml data type is a representation of a sequence of parsed XML. It can contain various elements such as XML elements, processing instructions, comments, and text that appear within an XML element.
  • Any XML value : xml
  • Constraint XML Type : xml <𝓣>

Here 𝓣 is subtype of xml:Element|xml:ProcessingInstruction|xml:Comment|xml:Text

  • XML Template
    e.g. xml `<name id="12">${text}</name>`
  • Lang Library : ballerina/lang.xml
  • Iterable 🔁

🛠  Array Type 

An Array type describes a set of list values. A list value is a container that keeps its members in an ordered list and the index of the member is used to access an member. The Array syntax specifies a type, and all members of the list must belongs to that type. The Array syntax is variable length (open) by default unless its length is specified.
  • Variable Length : 𝓣[]
  • Fixed Length : 𝓣[𝓲]
  • Fixed Length - Inferred : 𝓣[*]
  • Multi Dimension : 𝓣[][][], T[𝓲][𝓳][𝓴]

Here 𝓣 is the member type, 𝓲, 𝓳 and 𝓴 are integer values(or integer constants)

  • List Constructor
    e.g. [1, 2, 3, 4], [[1,2], [3,4]]
  • Lang Library : ballerina/lang.array
  • Iterable 🔁

🛠  Tuple Type 

Like an Array, a Tuple type describes a set of list values. However, The Tuple syntax specifies a set of types separately for each member of the list. The Tuple syntax is fixed in length by default, but it can be variable length with a rest field.
  • Fixed Length : [𝓣, 𝓢]
  • Open Length : [𝓣, 𝓡...]

Here 𝓣 and 𝓢 are member types. 𝓡 is the rest field type.

  • List Constructor
    e.g. [1, 2, 3, 4]
  • Lang Library : ballerina/lang.array
  • Iterable 🔁

🛠  Map Type 

A Map type describes mapping values. A mapping value is a container where each member has a unique string key, which is used to look up a member. The Map syntax specifies a type, and all members of the mapping value must belong to that type.
  • map<𝓣>

Here 𝓣 is the member type.

  • Mapping Constructor
    e.g. {𝓪 : 1, 𝓫 : true}
    Here 𝓪 and 𝓫 are filed names.
  • Lang Library : ballerina/lang.map
  • Iterable 🔁

🛠  Record Type 

Similar to a Map, a Record type describes mapping values too. Similar to Tuple syntax, The Record syntax specifies separate type for each member of the mapping value. The term field is used to represent both the type of the member and its key together.
  • Open Record - Inclusive (Open to anydata)
    record { 𝓣 𝓪; 𝓢 𝓫; }
  • Open Record - Exclusive
    record {| 𝓣 𝓪; 𝓡...; |}
  • Close Record - (No Rest Field)
    record {| 𝓣 𝓪; 𝓢 𝓫; |}
  • Optional Field
    record {| 𝓣 𝓪?; |}
  • Default Values
    record {| 𝓣 𝓪 = 𝓮𝔁𝓹; |}
  • Readonly Field
    record {| readonly 𝓣 𝓪?; |}
  • Record Inclusion
    record {| *𝓣; |}

Here 𝓣 and 𝓢 are member types with their field names 𝓪 and 𝓫. 𝓡 is the rest member type. 𝓮𝔁𝓹 is an expression of Type 𝓣.

  • Mapping Constructor
    e.g. {𝓪 : 1, 𝓫 : true}
    Here 𝓪 and 𝓫 are filed names.
  • Lang Library : ballerina/lang.map
  • Iterable 🔁

🛠  Table Type 

A Table type represents a tabular data structure, where members are mapping values. The table keeps members in order, but unlike list values, you cannot access members using an index. The Table syntax defines a key to access its members.
  • Table with Individual Keys
    table <𝓣> key (𝓪, 𝓫)
  • Table with Key Constrain 𝓢
    table <𝓣> key <𝓢>
  • Keyless Table
    table <𝓣> or table <𝓣> key <never>
  • Any Table Value
    table

Here 𝓣 and 𝓢 are Record types. 𝓪 and 𝓫 are table keys which are required and readonly field names with subtype of anydata of the 𝓣.

  • Table Constructor, e.g.
    table [{𝓪 : 1, 𝓫 : true}, {𝓪 : 2, 𝓫 : false}]
    Here 𝓪 and 𝓫 are filed names.
  • Lang Library : ballerina/lang.table
  • Iterable 🔁

🛠  Type References 

You can refer to a type definition in the current or different module using a type reference.
  • Type definition 𝓐 in the current module : 𝓐
  • Type definition 𝓐 in module 𝓶 : 𝓶:𝓐
  • Type Definition
    type 𝓐 𝓣;
    Here 𝓣 is the type and 𝓐 is the type definition name.

🛠  Singleton - Single Value Type 

A singleton type represents a single shape value, such as compile-time constants, literal values, etc.
  • Nil Literal, e.g. (), null
  • Boolean Literal, e.g. true, false
  • Integer Literal, e.g. 123, -123
  • Floating Point Literal, e.g. 12.34, -12.34
  • String Literal, e.g. Hello
  • Readonly Constant References**
  • Define Type definitions
    type Age "age";
  • Variable Definition
    10 ten = 10;

🛠  Error Type 

Errors values allow developers to communicate information about errors that have occurred in a program. Error values live separately from other values, which allows language constructs to handle errors differently.
  • Error type with detail mapping 𝓣
    error <𝓣>
  • Any Error : error
  • Distinct Error type with detail mapping 𝓣
    distinct error <𝓣>

Here 𝓣 is a subtype of map<error:Cloneable>

  • Error Constructor
    e.g. error ("error message")
  • Error values are immutable.
  • Lang Library : ballerina/lang.error

🛠  Function Type 

A function is a value that references a function. Like other values, it can be passed to or returned from a function, stored in a variable or data structure, etc. A function value can be executed by passing an argument list as input, similar to a normal function call.
  • Function Type with Parameters and Return Type
    function (𝓣 𝓪, 𝓢 𝓫) returns 𝓡
  • Any Function
    function

Here 𝓣 and 𝓢 are parameter types with names; 𝓪 and 𝓫. 𝓡 is the return type

  • Anonymous Function Expression
  • Lang Library : ballerina/lang.function

🛠  Object Type 

An Object value encapsulates data and functions that operate on data. They are composed of named members which are either a field or a method. In Ballerina, network primitives such as client, service, and listeners are built using object types. Client and service objects are two specific kinds of objects that are used for network communication. Additionally language defines a set of built-in object types (i.e. Listener, Iterator, etc.) that are used for various language constructs.
  • Object with a Field 𝓪
    object { 𝓣 𝓪; }
  • Object with a Method 𝔁
    object { function 𝔁 (𝓣 a, 𝓢 𝓫) returns 𝓡; }
  • Object Inclusion : object { *𝓞; }
  • Any Object : object { }
  • Client Object : client object { }
  • Service Object : service object { }
  • Remote and Resource methods

Here 𝓣, 𝓢 and 𝓡 are types, 𝓞 is an object type, 𝔁, 𝓪 and 𝓫 are identifiers.

  • Object Constructor
  • Lang Library : ballerina/lang.object

🛠  Future Type 

A future represents a value that will return by a named worker. A future value can be used to execute asynchronous tasks in a concurrent manner.
  • future <𝓣>
  • Any Future Value : future

Here 𝓣 is future value type.

  • Start Action
  • Lang Library : ballerina/lang.future

🛠  Stream Type 

A stream is a value that can produce a sequence of values. Streams are used by many standard library functions to present a sequence of values sourced from a file, message queue, or database table, etc. It is composed of two primitive operations: “next” and “close”. The “next” operation retrieves values from the sequence (similar to the “next” method of an iterator), while the “close” operation closes the stream, releasing its resources.
  • stream <𝓣>
  • Query Expression, New Expression
  • Lang Library : ballerina/lang.stream

🛠  Typedesc Type 

A type descriptor value is an immutable, unchanging representation of a resolved type descriptor in your program. When referencing an identifier defined by a type definition in an expression, it will produce a type descriptor value.
  • Type Descriptor with a constraint
    typedesc <𝓣>
  • Any Type Descriptor typedesc
  • typeof Operator
  • Lang Library : ballerina/lang.typedesc

🛠  Native Data - Handle Type 

A handle value is essentially a pointer to a native data that managed outside of the Ballerina value space, such as Java Object or C pointer. They are also useful for interacting with non-Ballerina code, allowing Ballerina code to interact with existing APIs or libraries that use other languages. Handle values are platform (i.e. jBallerina) dependent and used with external function bodied functions. Handle values are immutable.
  • handle
  • jBallerina Library
    • ballerina/jballerina.java
    • ballerina/jballerina.java.arrays

🛠  Any Type 

The any type describes a type consisting of all values other than errors. This type is useful when the data type is unknown or when a value needs to change type at runtime. However, using the any type makes the code less readable and is thus considered an anti-pattern.
  • any
  • Related Lang Library : ballerina/lang.value

🛠  Any Data Type 

The anydata type describes all plain data values in Ballerina. It contains a shape only if it is the shape of a plain data value.
  • anydata
  • Related Lang Library : ballerina/lang.value
  • anydata type using an union type
    () | boolean | int | float | decimal | string | xml | regexp:RegExp | anydata[] | map<anydata> | table<anydata>

🛠  JSON Type 

The json type is a built-in type in Ballerina used to represent JSON data, including values such as null, numbers, strings, booleans, JSON arrays, and JSON objects. Ballerina provides easy-to-use language features like lax typing, navigation, and libraries to work with JSON data. It is always recommended to use the built-in data-binding feature with JSON values for better compile-time validation.
  • json
  • Related Lang Library : ballerina/lang.value
  • json type using an union type
    () | boolean | int | float | decimal | string | json[] | map<json>

  • JSON supports lax static typing
🛠  Byte Data Type 

byte type is built-in union type, that includes int values between 0 and 255 inclusive. (0 ≤ values ≤ 255).
  • byte
  • Base16/Base64 Byte Array Literals
  • Type byte’s value space contains only int values, it is a subtype of the int
  • Type byte and built-in subtype int:Unsigned8 are equivalent.
  • Related Lang Library : ballerina/lang.int

🛠  No Shape - Never Type 

The never type describes the type that does not contain any shape. Hence there is no value in Ballerina values space that belongs to the never type. This is different from nil, where nil describes the absence of any value and it has a single value called nil. The never type is useful for indicating that a function will never return a value.
  • never
  • Any type 𝓣, the type 𝓣|never is the same as 𝓣.

🛠  Immutable - Readonly Type 

The Readonly type describes all inherently immutable values. A value belongs to the readonly type if its read-only bit is on. For example, The types nil, boolean, int, float, decimal, string, error, function, typedesc, and handle always have their read-only bit on and are a part of the readonly type. Other types such as xml, list, mapping, table, and object only belong to readonly only if their read-only bit is on.
  • readonly
  • Related Lang Library : ballerina/lang.value
