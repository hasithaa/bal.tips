Any Type

The any type describes a type consisting of all values other than errors. This type is useful when the data type is unknown or when a value needs to change type at runtime. However, using the any type makes the code less readable and is thus considered an anti-pattern.
  • any
  • Related Lang Library : ballerina/lang.value

Any Data Type

The anydata type describes all plain data values in Ballerina. It contains a shape only if it is the shape of a plain data value.
  • anydata
  • Related Lang Library : ballerina/lang.value
  • anydata type using an union type
    () | boolean | int | float | decimal | string | xml | regexp:RegExp | anydata[] | map<anydata> | table<anydata>

JSON Type

The json type is a built-in type in Ballerina used to represent JSON data, including values such as null, numbers, strings, booleans, JSON arrays, and JSON objects. Ballerina provides easy-to-use language features like lax typing, navigation, and libraries to work with JSON data. It is always recommended to use the built-in data-binding feature with JSON values for better compile-time validation.
  • json
  • Related Lang Library : ballerina/lang.value
  • json type using an union type
    () | boolean | int | float | decimal | string | json[] | map<json>

  • JSON supports lax static typing

No Shape - Never Type

The never type describes the type that does not contain any shape. Hence there is no value in Ballerina values space that belongs to the never type. This is different from nil, where nil describes the absence of any value and it has a single value called nil. The never type is useful for indicating that a function will never return a value.

Immutable - Readonly Type

The Readonly type describes all inherently immutable values. A value belongs to the readonly type if its read-only bit is on. For example, The types nil, boolean, int, float, decimal, string, error, function, typedesc, and handle always have their read-only bit on and are a part of the readonly type. Other types such as xml, list, mapping, table, and object only belong to readonly only if their read-only bit is on.
  • readonly
  • Related Lang Library : ballerina/lang.value
